## Business Goal

Build the APOS Core Engine: the core intelligence layer that plans, analyzes,
loads knowledge, generates (in memory), validates, and prepares deployments for
existing websites. It is not a website generator and not a runtime demo. The
first supported vertical is Healthcare (siud.org), but the architecture stays
generic.

---

## Scope

Create packages/core with independent modules: planner, analyzer, knowledge,
generator, validator, deployment, plus shared models and port types.

Expose a public API from index.ts, including a CoreEngine facade and a
composition root.

---

## Out of Scope

LLM integration.

Hardcoded business logic or business facts.

Website generation / file writing.

Any modification of the target project.

## Notes

Generic by construction: no vertical-specific facts live in the engine. Business
knowledge (including the Healthcare/siud.org pack) is supplied at runtime through
a `VerticalPackRepository`, so adding an industry never changes the engine.

Deferred intelligence is expressed as interfaces only: `RequestInterpreter`
(natural-language understanding) has no concrete implementation, and the
`Generator` is an honest structural skeleton returning empty output. Both are
stable seams for future LLM-backed adapters.

Module independence: each module imports only from `models/` and `types/`, never
from a sibling module. `src/index.ts` is the single composition root and the
`CoreEngine` facade depends solely on port interfaces.

Safety: the deployment stage only *describes* file operations; nothing on disk is
created, modified, or deleted, and no file is written anywhere in the engine.
