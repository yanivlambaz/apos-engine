## Business Goal

Build the APOS Runtime Core: a modular execution runtime that coordinates work
without embedding business logic or calling any language model.

---

## Scope

Expand packages/runtime into independent modules.

Implement the Orchestrator, Dispatcher, Document Loader, Workflow, Execution
Report and shared types.

Define clean interfaces (ports) between modules.

---

## Out of Scope

Business logic.

LLM integration.

Generators, validators and pattern selection.

Document parsing or knowledge extraction.

## Notes

Module boundaries enforce independence: concrete modules import only from
`types/`, and the orchestrator depends on port interfaces (`DispatcherPort`,
`WorkflowPort`, `ExecutionReportBuilderPort`) rather than concrete classes.

`src/index.ts` is the single composition root; it wires concrete modules and
shares one `IdFactory` across them.

The Document Loader is deliberately not a dependency of the Orchestrator:
document loading and execution coordination are independent responsibilities.
