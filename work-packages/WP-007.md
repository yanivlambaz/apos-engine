# WP-007

Title

Authority Intelligence Engine

Status

In Progress

Priority

Critical

---

## Business Goal

Implement the APOS Authority Intelligence Engine: the subsystem that decides
whether a completed Expert Conversation should become a permanent Authority
Asset, per SEARCH_INTENT_SYSTEM.md. It never generates content and never
generates pages — it only produces an Authority Recommendation that a later
workflow, and eventually the Generators, may act upon.

---

## Scope

Create `packages/core/src/intelligence/` modules:
`AuthorityIntelligenceEngine`, `SearchIntentAnalyzer`, `AuthorityScorer`,
`CoverageAnalyzer`, `DuplicateDetector`, `AuthorityRecommendation`,
`RecommendationRepository`, and an `index.ts` barrel.

Export the complete Authority Intelligence Engine from
`packages/core/src/index.ts`.

---

## Out of Scope

LLM implementation.

Page generation.

Content generation.

Deployment.

Website file inspection (Coverage Analyzer is interfaces only).

Scoring algorithms (Authority Scorer is architecture only).

AI reasoning for duplicate matching.

Healthcare-specific or any vertical-specific logic.

## Deliverables

- `packages/core/src/intelligence/SearchIntentAnalyzer.ts` — identifies
  Search Intent, Search Intent Type, Search Domain, Search Entity and User
  Intent from a completed conversation; `SearchIntentClassifierPort` is the
  AI-provider seam, with an honest fallback (the user's own first message,
  everything else unclassified) when none is configured.

- `packages/core/src/intelligence/CoverageAnalyzer.ts` — `CoverageAnalyzerPort`
  and a default `UnknownCoverageAnalyzer` that reports "unknown" without
  inspecting any website files.

- `packages/core/src/intelligence/DuplicateDetector.ts` — `DuplicateDetectorPort`
  and a default `NoOpDuplicateDetector` that reports no duplicates without
  reasoning over the Knowledge Object registry.

- `packages/core/src/intelligence/AuthorityScorer.ts` — `AuthorityScorerPort`
  over the six evaluation dimensions (Search Intent Match, Authority
  Expansion, Existing Coverage, Commercial Opportunity, Conversation Quality,
  Confidence) and a default `ZeroAuthorityScorer` with no combining algorithm.

- `packages/core/src/intelligence/AuthorityRecommendation.ts` — the
  `AuthorityRecommendation` record (Decision, Confidence, Reasoning, Related
  Knowledge Objects, Related Conversations, Search Intent, Suggested
  Pattern) and the four allowed `AuthorityDecision` values.

- `packages/core/src/intelligence/RecommendationRepository.ts` —
  `InMemoryRecommendationRepository`, enforcing one recommendation per
  conversation.

- `packages/core/src/intelligence/AuthorityIntelligenceEngine.ts` — public
  interface. `evaluate(conversation)` returns exactly one
  `AuthorityRecommendation`. Defines `AuthorityDecisionPolicyPort` as the
  decision-making seam, with a default `ConservativeAuthorityDecisionPolicy`
  (always `IGNORE`) since no real evidence exists without the other seams
  connected.

- `packages/core/src/intelligence/index.ts` — public barrel.

- Authority Intelligence Engine exports added to `packages/core/src/index.ts`.

## Notes

Conservative default decision: `AuthorityIntelligenceEngine.evaluate` always
routes through an `AuthorityDecisionPolicyPort`. The shipped
`ConservativeAuthorityDecisionPolicy` always returns `IGNORE` with confidence
0, because in this foundation none of `SearchIntentClassifierPort`,
`CoverageAnalyzerPort`'s real inspection, `DuplicateDetectorPort`'s real
reasoning, or `AuthorityScorerPort`'s real scoring exist yet — recommending
anything else would mean inventing authority expansion from no evidence,
which SEARCH_INTENT_SYSTEM.md's Core Principle explicitly rejects ("the goal
is not to create more pages").

Three independent, swappable seams: `SearchIntentClassifierPort`,
`CoverageAnalyzerPort`, `DuplicateDetectorPort`, `AuthorityScorerPort`, and
`AuthorityDecisionPolicyPort` are all separate interfaces so a future
implementation can replace them one at a time (e.g. plug in a real Coverage
Analyzer against a website snapshot without touching duplicate detection or
scoring).

`AuthorityScore` intentionally has no "overall" field: combining the six
dimensions into one number is itself a scoring algorithm, which this work
package explicitly excludes ("do not implement scoring algorithms").

`relatedConversationIds` currently contains only the evaluated conversation
itself; aggregating other historical conversations that share the same
Search Intent requires a cross-conversation lookup that is not implemented in
this foundation.

`AuthorityIntelligenceEngine.evaluate` throws if given a conversation whose
state is not `closed`, mirroring the same structural guard used by
`ConversationRepository` (WP-006) for "completed conversations only".

This engine consumes `Conversation` from the Expert Engine (WP-006) and
`ObjectRegistryReader` from the Object Model (WP-004) directly, since both
are established, stable read-only contracts; no behavior in either module was
modified.
