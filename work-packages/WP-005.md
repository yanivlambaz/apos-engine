# WP-005

Title

Knowledge Acquisition Engine

Status

In Progress

Priority

Critical

---

## Business Goal

Implement the APOS Knowledge Acquisition Engine: the subsystem responsible for
discovering, loading, indexing and maintaining the project's knowledge
sources. It prepares the trusted knowledge base the rest of APOS consumes. It
does not answer questions and does not generate content.

---

## Scope

Create `packages/core/src/knowledge/` modules: `KnowledgeManager`,
`KnowledgeSource`, `KnowledgeIndex`, `KnowledgeLoader`, `KnowledgeRegistry`,
`SourcePriority`, and a consolidated `index.ts` barrel.

Support Project Knowledge directory discovery, Vertical Pack discovery, and an
interface-only External Knowledge source.

Implement the four-tier source priority model from KNOWLEDGE_SYSTEM.md.

Export the complete Knowledge Engine from `packages/core/src/index.ts`.

---

## Out of Scope

Document parsing / content extraction.

AI-based concept extraction.

LLM integration.

Web search implementation.

Content generation.

Deployment logic.

Healthcare-specific or any vertical-specific logic.

---

## Dependencies

`docs/KERNEL/KNOWLEDGE_SYSTEM.md`.

`packages/core` from WP-003 (Core Engine foundation, `VerticalPackRepository`

## Deliverables

- `packages/core/src/knowledge/SourcePriority.ts` — the four-tier priority
  model and comparison helpers.

- `packages/core/src/knowledge/KnowledgeSource.ts` — `KnowledgeSource` /
  `ExternalKnowledgeSource` interfaces, `ProjectKnowledgeDirectorySource`
  (filesystem discovery of `knowledge/`), `VerticalPackKnowledgeSource`
  (discovery over an existing `VerticalPackRepository`).

- `packages/core/src/knowledge/KnowledgeLoader.ts` — loads a source by
  invoking discovery only.

- `packages/core/src/knowledge/KnowledgeIndex.ts` — in-memory structures for
  future extracted knowledge (`KnowledgeIndexEntry`, relationship linking).

- `packages/core/src/knowledge/KnowledgeRegistry.ts` — tracks source id,
  type, priority, status, last indexed time and document count; rejects
  duplicate source ids.

- `packages/core/src/knowledge/KnowledgeManager.ts` — public entry point
  coordinating discovery, loading, indexing and registry updates.

- `packages/core/src/knowledge/index.ts` — consolidated barrel (folds in the
  existing `KnowledgeBase`/`InMemoryVerticalPackRepository` from WP-003).

- Knowledge Engine exports added to `packages/core/src/index.ts`.

## Notes

`ProjectKnowledgeDirectorySource` performs real recursive filesystem
discovery (via `node:fs/promises`) of supported file types (PDF, DOCX, MD,
TXT, HTML, CSV, JSON) but never reads or parses file contents — only the
architecture for acquisition exists, per the requirement.

`VerticalPackKnowledgeSource` reuses the existing `VerticalPackRepository`
port from WP-003 rather than duplicating pack-loading logic; it exposes each
business object as one discoverable document reference without interpreting
its attributes.

`ExternalKnowledgeSource` is an interface only, with no concrete
implementation, per the "no web search implementation" and "interfaces only"
requirements.

`KnowledgeIndex` entries carry empty `concepts` and `relatedEntryIds` by
default; nothing in this engine ever populates them, since that requires
extraction/AI capability not yet implemented.

The pre-existing `KnowledgeBase` (Vertical Pack loading for the Core Engine
pipeline, from WP-003) was consolidated into a single `knowledge/index.ts`
barrel alongside the new engine to avoid duplicate export paths; its behavior
is unchanged.
