# WP-004

Title

APOS Object Model

Status

In Progress

Priority

Critical

---

## Business Goal

Design and implement the generic APOS Knowledge Object Model. APOS manages
Knowledge Objects; pages are only projections built from those objects. The
model must support Healthcare as the first vertical while remaining fully
generic for future verticals.

---

## Scope

Create `packages/core/src/objects/`.

Implement the abstract base Object and concrete Knowledge Object classes:
Service, Guide, FAQ, Category, Location, Article and Procedure.

Implement generic relationship support.

Implement an Object Registry for registering, retrieving, querying and
structurally validating objects.

Export the complete Object Model from `packages/core/src/index.ts`.

---

## Out of Scope

Business validation.

Healthcare-specific relationships or facts.

React, Next.js, page generation, file generation, deployment logic and LLM
integration.

## Notes

Relationships are represented as data through `ObjectRelationship`, with a
target ID, optional target type hint, caller-defined relationship type and
metadata. The registry validates structure and target integrity only; it does
not decide which object types may relate to one another.

The base class is exported as `Object` to match the APOS domain language. Module
consumers may alias it on import if they want to avoid confusion with the
JavaScript global `Object`.
