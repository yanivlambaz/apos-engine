# Purpose

This document defines how information must be organized inside every APOS project.

Information Architecture is not navigation.

Information Architecture is the logical organization of knowledge.

Navigation is only one way to expose it.

A poor Information Architecture cannot be fixed with better design.

# Navigation Is Not Architecture

Navigation exists for humans.

Architecture exists for everyone.

A visitor may never open the menu.

Google never clicks.

LLMs never use menus.

Therefore:

Navigation must expose architecture.

Not replace it.

# Internal Linking Philosophy

Internal links should represent knowledge relationships.

Never random links.

Never links only for SEO.

Every internal link answers:

"What should the user learn next?"

# Page Independence

Every page should be understandable independently.

No page should require another page to explain its purpose.

---

# Knowledge Independence

Knowledge should survive redesigns.

Changing navigation must not require rewriting knowledge.

Changing design must not require rewriting knowledge.

---

# Growth Model

Every new page must strengthen the platform.

Never create isolated pages.

New content should always connect to existing knowledge.

---

# Structural Integrity

The architecture should continue working even if:

Homepage changes.

Navigation changes.

Design changes.

Technology changes.

Framework changes.

Architecture must outlive implementation.

---

# Validation

Architecture is considered correct when:

Users easily understand the platform.

Google understands the platform.

LLMs understand the platform.

Internal links are meaningful.

Knowledge is complete.

Relationships are explicit.

Growth becomes easier over time.

---

# Anti-Patterns

Do not build isolated landing pages.

Do not build disconnected articles.

Do not build city pages without local value.

Do not create duplicate knowledge.

Do not organize content around marketing campaigns.

Do not let navigation define architecture.

---

# Final Principle

Think like a librarian.

Not like a marketer.

Not like a designer.

Organize knowledge first.

Everything else becomes dramatically easier.
