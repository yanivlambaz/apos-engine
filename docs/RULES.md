# RULES.md
# APOS Engineering Rules

Version: 1.0

Status: STABLE

Classification: MANDATORY

---

# Purpose

These rules are mandatory.

Every AI Agent.

Every Developer.

Every Contributor.

Must follow them.

These rules override preferences.

---

# Rule 1

Protect the architecture.

Never implement a solution that weakens the architecture for short-term convenience.

---

# Rule 2

Reuse before creating.

Before creating

Component

Object

Pattern

Generator

Validator

Workflow

Search the project.

Reuse if possible.

---

# Rule 3

Never duplicate.

Never duplicate

Business Logic

Components

Pages

Objects

Utilities

Configuration

Documentation

Knowledge

---

# Rule 4

Business knowledge belongs only inside Vertical Packs.

Never place industry-specific knowledge inside the Kernel.

---

# Rule 5

Never hardcode.

Configuration belongs in configuration.

Content belongs in Objects.

Rules belong in the Kernel.

---

# Rule 6

One responsibility.

Every file should have one clear responsibility.

Every module should have one purpose.

---

# Rule 7

Readable before clever.

Always prefer

Simple

Readable

Maintainable

Deterministic

code.

---

# Rule 8

Every change must improve the project.

Never create technical debt knowingly.

Leave the codebase cleaner than before.

---

# Rule 9

Never bypass validation.

Validation is mandatory.

Warnings may pass.

Errors may not.

Critical issues block release.

---

# Rule 10

Never invent facts.

Never fabricate

Statistics

Reviews

Regulations

Medical advice

Financial advice

Legal advice

Business data

---

# Rule 11

Update documentation.

Whenever architecture changes,

documentation changes.

Whenever behavior changes,

documentation changes.

---

# Rule 12

Optimize globally.

Never optimize one page at the expense of the whole platform.

---

# Rule 13

Think in systems.

Never solve isolated problems.

Every decision affects the platform.

---

# Rule 14

Preserve consistency.

Naming

Structure

Patterns

Style

Architecture

must remain consistent.

---

# Rule 15

Stop when information is missing.

Ask.

Report.

Wait.

Never guess.

---

# Rule 16

Prefer composition.

Never create large monolithic modules.

Build reusable building blocks.

---

# Rule 17

Security first.

Protect user data.

Protect credentials.

Protect secrets.

Never expose sensitive information.

---

# Rule 18

Accessibility is mandatory.

Never sacrifice accessibility for appearance.

---

# Rule 19

Performance is architecture.

Fast software is part of correctness.

---

# Rule 20

The project is more important than the task.

Optimize for

future maintainability,

future contributors,

future AI Agents,

future scalability.

Never optimize only for the current request.

---

# Final Rule

If two possible implementations exist,

choose the one that

reduces complexity,

improves consistency,

improves reuse,

and makes the project easier to maintain in five years.
