# APOS - PATTERNS

Version: 1.0

Status: STABLE

Classification: CORE KERNEL

---

# Purpose

Patterns define how Objects become Pages.

Objects contain information.

Patterns define presentation.

Patterns never contain business knowledge.

Patterns are reusable across every Vertical Pack.

---

# Rule

Every page generated by APOS MUST use one Pattern.

No page may exist without a Pattern.

# Universal Rules

Every Pattern MUST

Contain exactly one H1.

Be mobile-first.

Support accessibility.

Support structured data.

Support internal linking.

Support AI readability.

Support SEO.

Support conversion.

Support future expansion.

---

# Forbidden

No duplicated sections.

No duplicated content.

No multiple primary CTAs.

No placeholder content.

No fake statistics.

No orphan pages.

✓ AI requirements pass

✓ Accessibility requirements pass
