# APOS Search Intent System

Version: 1.0

Status: CORE

Classification: KERNEL

---

# Purpose

The Search Intent System defines how APOS decides whether a completed expert conversation should become a new content asset.

APOS never creates pages simply because a conversation exists.

APOS creates content only when the conversation expands the Authority Platform.

---

# Core Principle

Every conversation represents a possible Search Intent.

The purpose of APOS is to determine whether that Search Intent deserves a permanent content asset.

The goal is not to create more pages.

The goal is to increase the Authority Platform's coverage.

---

# Search Intent

A Search Intent represents a question that users actively search for.

Examples

"How much does National Insurance pay for a caregiver?"

"Where is the National Insurance office in Hadera?"

"Does Clalit provide home nursing after surgery?"

Search Intent may be

Commercial

Informational

Navigational

Transactional

Every Search Intent may become a permanent knowledge asset.

---

# Content Opportunity Evaluation

Every completed conversation must be evaluated.

The evaluation produces one of four decisions.

IGNORE

UPDATE EXISTING PAGE

MERGE WITH EXISTING CONTENT

CREATE NEW INTERNAL PAGE

No other outcome is allowed.

---

# Evaluation Criteria

The engine should evaluate:

## Search Intent Match

Does the conversation contain recognizable search intent?

## Vertical Relevance

Does the conversation belong to the project's Vertical?

## Authority Contribution

Will this conversation expand the Authority Platform?

## Existing Coverage

Does the website already answer this question?

## Duplicate Detection

Is there already a page answering substantially the same intent?

## Commercial Value

Can this topic naturally connect users to relevant services?

Commercial value is never the only decision factor.

## Conversation Quality

Was the expert able to answer confidently using approved knowledge?

Only approved knowledge may be used.

# Decision Rules

APOS should prefer

UPDATE

over

CREATE

whenever an existing page can reasonably satisfy the intent.

Create a new page only when the Search Intent introduces meaningful new authority.

---

# Internal Page Format

When APOS decides to create a new page,

the page should be built as an Expert Conversation.

Typical structure

Question

↓

Expert Answer

↓

Knowledge Sources

↓

Related Questions

↓

Related Services

↓

Related Knowledge

↓

FAQ

↓

Call To Action

The exact layout belongs to the Pattern System.

---

# Search Engines

Every generated page should be optimized for

Traditional Search Engines

AI Search Engines

Large Language Models

Answer Engines

without changing the original expert answer.

---

# Continuous Growth

Every approved conversation strengthens the Authority Platform.

Over time,

thousands of expert conversations become structured knowledge assets.

The Authority Platform continuously expands without changing the APOS Engine.

---

# Final Rule

APOS never creates pages to increase page count.

APOS creates knowledge assets to expand Search Intent coverage and Authority.

Page count is a consequence.

Authority growth is the objective.
