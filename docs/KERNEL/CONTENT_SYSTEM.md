# APOS - CONTENT SYSTEM

Version: 1.0

Status: STABLE

Classification: CORE KERNEL

---

# Purpose

This document defines the content standards for every asset generated by APOS.

Content exists to educate.

Trust is earned through clarity.

Conversion is the result of understanding.

Every piece of content must improve the Knowledge Graph.

---

# Content Philosophy

Write for people first.

Optimize for AI second.

Optimize for search engines third.

Never reverse this order.

---

# Primary Objectives

Every content asset must

Answer a real question.

Reduce uncertainty.

Increase trust.

Support decision making.

Connect to related knowledge.

Support business objectives naturally.

---

# Writing Principles

Use simple language.

Use short paragraphs.

Use descriptive headings.

Use practical examples.

Use bullet lists where appropriate.

Use tables when comparison improves understanding.

Avoid filler.

Avoid repetition.

Avoid marketing exaggeration.

Avoid unnecessary jargon.

---

# Tone of Voice

Professional.

Helpful.

Confident.

Transparent.

Educational.

Objective.

Respectful.

Never aggressive.

Never manipulative.

Never fear-based.

---

# Reading Experience

What

Why

Who

within the first minute.

---

# Introductions

Every introduction should

Explain the topic.

Set expectations.

Define who the content is for.

Maximum recommended length

200 words.

---

# Headings

Every heading should

Describe the section.

Contain meaningful language.

Be understandable without surrounding context.

Avoid vague headings.

---

# Paragraphs

Maximum

4 sentences.

Prefer

2–3 sentences.

Avoid large text blocks.

---

# Lists

Use lists when

Explaining steps.

Comparing options.

Presenting requirements.

Summarizing benefits.

Avoid long unstructured paragraphs.

---

# Tables

Use tables for

Comparisons.

Pricing.

Requirements.

Timelines.

Eligibility.

Never use tables for decoration.

---

# Examples

Every major concept should include practical examples whenever appropriate.

Examples should reflect real-world situations.

Never invent facts.

---

# Definitions

Every important term should be clearly defined.

Definitions should appear before advanced explanations.

Link to glossary where available.

---

# FAQ

Every major page should contain FAQs.

Questions should come from real user intent.

Avoid duplicate questions across the platform.

Reuse FAQ Objects whenever possible.

---

# Calls To Action

CTAs should appear after value has been delivered.

Every CTA should explain

What happens next.

Expected response.

Available contact methods.

Avoid pressure tactics.

---

# Trust Signals

Content may include

Experience.

Professional credentials.

Transparent process.

Verified reviews.

Real timelines.

Real limitations.

Never fabricate trust.

---

# Content Freshness

Every asset includes

Created Date

Last Updated

Review Frequency

Content Owner

Review outdated content regularly.

---

# AI Readability

Every page should explicitly answer

What is it?

Why is it important?

Who is it for?

How does it work?

What should the user do next?

Avoid requiring inference.
